# CR-UNITS-PER-LOT — Historical Metric Invalidation & Legacy Freeze

**Gate:** `ALL_LEGACY_SHADOW_PNL = INVALID_FOR_DECISION`
**Date:** 2026-06-23
**Status:** FROZEN — all prior shadow campaign metrics declared invalid for live-readiness decisions

---

## 1. Root Cause

All prior shadow campaigns used `units_per_lot = 100000` (hardcoded forex standard) instead of resolving `contract_size` from the broker runtime snapshot via `ContractSpec`. For XAUUSD (contract_size=100), position sizing was **1000x inflated** — a 0.1 lot strategy was executed as 100 lots, producing P&L figures with no relation to reality.

Broker symbol snapshots were available in every shadow run but the contract specification was never read. The `ContractSpecResolver` introduced by this CR closes that gap.

## 2. Invalidation Labels

Every prior metric is assigned one of three labels:

| Label | Meaning |
|---|---|
| `INVALID_FOR_DECISION` | Cannot be used for any live-readiness, funding, or risk decision |
| `SIMULATED_ONLY` | Valid as a simulated/educational figure under the old sizing model; no predictive power |
| `PARTIALLY_VALID` | Useful as operational telemetry but not for P&L or risk inference |

## 3. Metric-by-Metric Classification

| Metric | Label | Rationale |
|---|---|---|
| **Gross P&L** | `INVALID_FOR_DECISION` | Sized with wrong contract_size; absolute P&L is meaningless |
| **Net P&L** | `INVALID_FOR_DECISION` | Same error compounded by commission on wrong lot sizes |
| **Expectancy** | `INVALID_FOR_DECISION` | Derived from P&L; inherits the full sizing error |
| **Profit factor** | `INVALID_FOR_DECISION` | Gross win / gross loss both scaled by wrong lot size; ratio is a coincidence |
| **Risk per trade** | `INVALID_FOR_DECISION` | $ risk computed against wrong notional exposure |
| **Win rate** | `SIMULATED_ONLY` | Directional correctness may hold, but magnitude is wrong; not predictive |
| **Max drawdown** | `INVALID_FOR_DECISION` | Drawdown in absolute $ is meaningless when position size was wrong |
| **Risk/reward ratio** | `INVALID_FOR_DECISION` | Ratio derived from wrong P&L values per trade |
| **Signal count** | `PARTIALLY_VALID` | Operational telemetry — valid as count of entries; no P&L dependency |
| **Uptime / connectivity** | `PARTIALLY_VALID` | Infrastructure test; independent of sizing |
| **Spread observations** | `PARTIALLY_VALID` | Raw market data; useful for spread analysis only |
